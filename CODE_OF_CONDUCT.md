# Code of Conduct

`cl-dataflow` aims to be a small, practical open source project. The expectation is simple:

- Be respectful and direct.
- Focus criticism on code, documentation, and design, not people.
- Assume good faith unless there is clear evidence otherwise.
- Keep review comments actionable.

## Enforcement

Project maintainers may ask contributors to edit or remove content that is hostile, discriminatory, or disruptive to collaboration.

## Scope

This applies to issues, pull requests, review comments, and any project discussion spaces that are explicitly associated with `cl-dataflow`.
